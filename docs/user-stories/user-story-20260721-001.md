## 🎯 User Story

**As a** 公開 Telegram 頻道的訂閱者  
**I want** 每天早上定時收到天氣預報通知，內容包含氣溫、降雨機率、風速等完整天氣資訊  
**So that** 我可以出門前掌握當日天氣狀況，做好行程規劃與衣物準備

---

## ✅ 驗收條件（Acceptance Criteria）

### AC-01：每日定時推播
**Given** 系統已正常運作且時間為每日 07:00（台灣時間）  
**When** GitHub Actions 排程觸發執行  
**Then** 系統自動抓取天氣預報並發送至指定的 Telegram 頻道

### AC-02：支援多資料源
**Given** 系統已設定資料來源  
**When** 使用者選擇 Open-Meteo（預設）或 CWB 氣象署 API  
**Then** 系統使用對應 API 取得天氣資料，並正常格式化輸出

### AC-03：天氣資訊完整性
**Given** 天氣資料抓取成功  
**When** 系統格式化訊息  
**Then** 通知內容包含：日期與星期、天氣現象（含 Emoji）、最高/低溫、體感溫度、降雨機率、風速風向，若資料源有提供則包含濕度與紫外線指數

### AC-04：城市可設定
**Given** 使用者想變更天氣查詢城市  
**When** 透過環境變數或 GitHub Actions 輸入參數  
**Then** 系統查詢指定城市的天氣預報

### AC-05：詳細程度可調
**Given** 使用者想調整通知詳簡度  
**When** 設定詳細程度參數（精簡/標準/完整）  
**Then** 系統依設定產出對應長度的天氣訊息

### AC-06：錯誤通知
**Given** 天氣 API 請求失敗或資料異常  
**When** 系統偵測到錯誤  
**Then** 系統發送錯誤通知至 Telegram，而非產生空白的氣象報告

### AC-07：手動觸發
**Given** 使用者想立即取得天氣預報  
**When** 在 GitHub Actions 頁面點擊「Run workflow」  
**Then** 系統立即執行並發送通知，不受排程時間限制

## ⚠️ 邊界案例（Edge Cases）

| 編號 | 情境 | 預期行為 |
|------|------|---------|
| EC-01 | API Key 無效或過期 | 系統記錄錯誤日誌，發送「API 授權失敗」通知 |
| EC-02 | 網路連線中斷 | 顯示明確的連線錯誤訊息，不產生空資料 |
| EC-03 | 輸入不存在的城市名稱 | 系統回傳「找不到該城市」錯誤，並列出可用的城市清單 |
| EC-04 | API 回傳資料格式異常 | 系統拋出解析錯誤，不發送不完整的氣象報告 |
| EC-05 | GitHub Actions 排程延遲（佇列等待） | 延遲不超過 30 分鐘，仍正常發送 |
| EC-06 | 使用者尚未將 Bot 加入頻道 | Telegram API 回傳 403 錯誤，系統記錄明確的權限錯誤 |

## ⚡ 非功能性需求（NFR）

| 類別 | 需求描述 | 驗收標準 |
|------|---------|---------|
| 效能 | API 請求回應時間 | 從觸發到發送完成 < 30 秒 |
| 可用性 | 排程準時率 | 每月延遲超過 30 分鐘的比例 < 5% |
| 可靠性 | 資料源備援 | Open-Meteo 失效時可快速切換至 CWB API |
| 可維護性 | 設定集中化 | 城市、時間、詳細程度皆透過環境變數或設定檔管理 |
| 安全性 | API Token 保護 | Token 存放於 GitHub Secrets，不寫入原始碼 |
| 成本 | 營運費用 | $0（GitHub Actions 免費額度 + 免費 API） |

## 🔗 依賴條件（Dependencies）

| 依賴項目 | 說明 | 狀態 |
|---------|------|------|
| Telegram Bot API | 需透過 @BotFather 建立 Bot 取得 Token | ⏳ 使用者自行申請 |
| Open-Meteo API | 免費天氣 API，免註冊免 Key | ✅ 無需申請 |
| CWB 氣象署 API | 台灣地區詳細天氣資料 | ⏳ 需至 opendata.cwb.gov.tw 申請 |
| GitHub Actions | CI/CD 排程執行環境 | ✅ Repo 建立後自動啟用 |
| Python 3.11+ | 執行環境 | ✅ 已選定 |
